• NEON MINING GAME – GDD v0.1 + Yol Haritası

  ## 1. Oyun Kimliği

  - Çalışma adı: Neon Mining Game
  - Tür: 2D top-down roguelike mining survival
  - Platform: PC (Steam), single-player
  - Hedef: 2–3 ayda playable prototype, 6–9 ayda
	yayınlanabilir sürüm
  - Hedef kitle: Vampire Survivors / Nova Drift seven
	arcade-roguelike oyuncuları

  ## 2. Core Oyun Döngüsü

  1. Asteroid bul ve mine et.
  2. Material topla.
  3. Düşman dalgası baskısı artar.
  4. Chain/multiplier ile ödülü büyüt.
  5. Perk seçerek build kur.
  6. DPS/mining gücü artar.
  7. Yeni zone açılır, risk/ödül yükselir.
  8. Run sonuna kadar hayatta kal.

  ## 3. Harita ve Zone Sistemi

  - Dünya yapısı: 5x5 zone grid = 25 zone
  - Her zone: ~16 ekran genişliği
  - Kamera: player merkezde, dünya/player’a göre akar
	hissi
  - Zone kuralları:
	  - Girişte başlangıç tehdit seviyesi atanır
	  - Zone içi mining yoğunluğu + enemy yoğunluğu
		farklılaşır
	  - Zone clear/ilerleme koşuluyla komşu zone açılır

  ## 4. Temel Sistemler

  - Mining:
	  - Asteroid HP, mining resist, drop table
	  - Kritik kazı / zincir kırmadan devam bonusu
  - Material ekonomisi:
	  - Run içi harcama (perk/sistem upgrade)
	  - Meta para (opsiyonel, erken aşamada şart değil)
  - Combat:
	  - Otomatik veya yarı otomatik atış
	  - Menzil, cooldown, projectile davranışları
  - Enemy Director:
	  - Süre + oyuncu gücüne göre spawn eğrisi
  - Perk sistemi:
	  - ~30 perk, 3 rarity bandı
	  - Sinerji odaklı (mining ↔ combat ↔ survivability)
  - Progression:
	  - 15–25 dk run akışı
	  - Orta oyunda tempo artışı, sonda yoğun baskı

  ## 5. İçerik Hedefi

  - Zone: 16
  - Enemy: 8–12
  - Asteroid: 6–8
  - Perk: ~30
  - Boss/elite: prototipte opsiyonel, full sürümde
	önerilir

  ## 6. Teknik Mimari (Godot/GDScript)

  - Ana sahneler:
	  - World
	  - Player
	  - EnemySpawner/Director
	  - ZoneManager
	  - MiningSystem
	  - PerkUI
	  - RunState
  - Veri odaklı yapı:
	  - Perk/Enemy/Asteroid değerleri Resource
		dosyalarında
  - Kaydetme:
	  - Prototipte run-state debug save
	  - Yayın öncesi settings + progression save

  ## 7. Denge Prensipleri

  - İlk 3 dakika: öğrenme + düşük baskı
  - 3–10 dakika: build kimliği oluşumu
  - 10+ dakika: karar baskısı ve risk yönetimi
  - Multiplier sistemi:
	  - Hızlı oynayanı ödüllendirir
	  - Hata durumunda reset/azalma ile gerilim üretir

  ## 8. Yol Haritası (Aşama Aşama)

  ### Aşama 1 – Pre-Production (Hafta 1–2)

  1. Tek sayfa “vision + pillar” dokümanı sabitle.
  2. Core sistem sınırlarını yaz: mining, combat, perk,
	 zone.
  3. Teknik spike: zone streaming + camera + shader grid
	 testi.
  4. Başarı kriterlerini tanımla:
	  - 20 dakikalık run oynanabiliyor mu
	  - Build farkı hissediliyor mu
	  - FPS stabil mi

  ### Aşama 2 – Playable Prototype (Ay 1–3)

  1. Vertical slice üret:
	  - 2x2 zone
	  - 3 asteroid tipi
	  - 4 enemy tipi
	  - 10 perk
  2. Tam core loop çalışsın:
	  - mine → drop → combat → perk → zone unlock
  3. UI temel:
	  - HP, material, multiplier, perk seçimi
  4. Teknik hedef:
	  - 60 FPS (orta seviye PC)
	  - Ölüm/yeniden run akışı
  5. Çıkış kriteri:
	  - Dış testte 10–15 dakikalık eğlenceli loop
		doğrulansın

  ### Aşama 3 – Production (Ay 3–6)

  1. İçeriği hedef sayılara genişlet:
	  - 25 zone, 8–12 enemy, 6–8 asteroid, ~30 perk
  2. Enemy Director + difficulty curve finalize.
  3. Sinerji/perk dengeleme iterasyonları.
  4. VFX/SFX pass (neon readability + hit feedback).
  5. Telemetry/debug panel:
	  - DPS, TTK, material/min, ölüm sebebi

  ### Aşama 4 – Beta ve Yayın Hazırlığı (Ay 6–9)

  1. Steam build pipeline + store hazırlığı.
  2. QA pass:
	  - crash, save, progression softlock
  3. Performans ve UX polish:
	  - input, menüler, tutorial ipuçları
  4. Denge freeze + son test turu.
  5. Release candidate + patch planı.

  ———
